# กติกาแพ้เจอแพ้ / ชนะเจอชนะ (PAIR_RESULT)

## ขอบเขตของหนึ่งบล็อก

เส้น `Game X → Game X+1` ที่กำหนดเป็น `PAIR_RESULT` จะทำให้ทั้งสองเกมเป็น Result block เดียวกัน เจ้าหน้าที่ต้องกรอกและตรวจผลครบทั้งสองเกมก่อนจึงจะ Publish และไปสร้าง Pairing ของ `Game X+2` ได้

รองรับจำนวนผู้เล่นทุกจำนวน รวมถึงจำนวนคี่ โดยระบบสร้างคู่บาย (ผู้เล่นหนึ่งคน อีกฝั่งว่าง)
และจัดกลุ่มเศษด้วย Swiss อัตโนมัติเมื่อกลุ่มท้ายมี 1 หรือ 2 คนเกินจากกลุ่มละ 4 คน
ห้ามกำหนด `PAIR_RESULT` ต่อกันมากกว่าหนึ่งเส้น

## การสร้างคู่ Game X+1

ระบบแบ่ง Match ของ Game X ทีละสองคู่ตามลำดับเลขคู่แข่งขัน เช่น คู่ 1–2, คู่ 3–4 แต่ละกลุ่มจึงมีผู้เล่นสี่คน

- ผู้ชนะของสองคู่ในกลุ่มเดียวกันพบกันในคู่บนของ Game X+1
- ผู้แพ้ของสองคู่ในกลุ่มเดียวกันพบกันในคู่ล่างของ Game X+1
- คู่ของ Game X+1 จะถูกสร้างเมื่อผล Game X ของทั้งสองคู่ในกลุ่มนั้นถูกบันทึกครบ
- ถ้า Game X เสมอ ระบบให้ผู้เล่นฝั่งแรกเข้าช่องบนและฝั่งที่สองเข้าช่องล่างอย่างคงที่ คะแนนสะสมยังเป็นเสมอ `+1 WP` ทั้งคู่
- ถ้าคู่ต้นทางเป็นคู่บาย จะกรอกผลได้เฉพาะให้ผู้เล่นที่อยู่ชนะเท่านั้น และช่องผู้แพ้ในเกมถัดไปจะว่าง
- กลุ่มท้ายที่มีผู้เล่นไม่ครบ 4 คนจะยังสร้าง Game X+1 ได้เสมอ โดยใช้คู่บายหรือ Swiss ตามจำนวนเศษ

เมื่อ Game X+1 มีผลแล้ว ห้ามแก้ผล Game X ในลักษณะที่ทำให้สมาชิกของคู่ Game X+1 เปลี่ยน หากคะแนนเปลี่ยนแต่ผู้ชนะ/ผู้แพ้เดิมไม่เปลี่ยนยังแก้ไขได้ และทุกการแก้ไขต้องมี Audit log

## การ Publish

- Finish ลงทะเบียน: public เห็นรายชื่อและ Ranking เริ่มต้น
- Finish Pairing Game X: public เห็นเฉพาะคู่ Game X โดยไม่เห็นคะแนน
- Finish ผล Result block: public เห็น Pairing และผลของ Game X กับ Game X+1 จาก immutable snapshot เดียวกัน
- จากนั้น `current_game` ข้ามไป `Game X+2` เพื่อให้เจ้าหน้าที่สร้าง Pairing ตามกติกาของเส้นก่อนหน้าเกมนั้น
